Project Description

This Project is designed for business analysis which provides useful information regarding Wines data. The data based on scapped information from two different wines websites and make a comparioson with existing database and provide price comparioson also provide the locations of the other stores selling the product and how close are those stores in 5 miles and 10 miles radius along with the locations on Map

The Road Map. Creating the Architecture of Project.

Before we got started, it was crutial to make a map of our road ahead.

What modules and code will be needed and what will we be using. What will be front end (HTML, CSS, JS), Flask as middle layer, whats on the back end (Mongo, MySql)

MySql . Python . SQL-Alchemy . BeautifulSoup(Scrape) . Google API . MongoDB . Flask . HTML . JavaScript . D3 . Leaflet

Database Migration. Microsoft SQL Server -> MySQL.

Database was originally in Microsoft SQL Server, so importing it to MySQL was the first step.

This required taking the Schema and Data and copying it to MySql.

Key Challanges: Before importing to MySql, we had to clean out "garbage" information. Otherwise data was not importing.

Garbage to Gold. The SQL-Alchemists.

Once the Database was set up, we had to connect to it using Python.

This allows us to referance the database, while Flask is running.

Key Challanges: Getting Automap to work streamlessly was a roadblock, we first created the classes for each table, but that was not sustainible as we bring in more tables. Solved this with some hard GoogleFu.

Scrape and Store. Beautiful Soup with a side of Mongo.

We knew that we wanted our application to be used for price comparison, so gaining prices from outside sources was key.

We chose to scrape Wine Searcher, the standard in alcoholic beverages for information on products

Key Challange: Many wines excluded some information, such as score and pairing. This prevented script from completing.... We also got our IP address blocked. So that was fun too.

Solution: Simple Try & Except methods worked for us. Oh and a VPN connection.

The Middle Brain. Flask with some Wine .. Wine+Flask.

Flask is 'pulling the strings', its the link to what information is in database and what we see on the front end.

Key Challanges: What routes will we need, and how is it going to get the information from front end, and link it to back end. We first tried having SQLite to pull and store information for quicker use. But it adds an extra step

Solution: Global Variables throughout Flask App. Once a form gets "POST"ed it sends the variable over to routes to run queries, which then get returned as json for future access.

Introduce Interactivity. JavaScripting with D3.

With a base HTML script, we went into the creating buttons, dropdowns, and filters to make the interactive dashboard.

Using forms, we were able to push information to Flask App, and then using JS create the content of the dashboard

Key Challange: In order for the search information to be used, it had to be "POST"ed to Flask, but that only happened on submiting. So finding a way to submit "onclick" was super crutial.

Solution: We were forced to go outside our class scope and use JQuery and Ajax. This allowed the POST functionality to occcur at the right times and create great speeds of information flow between user input, Flask, and backend querying.

Graphing the Information 1. Marked on Leaflet.

In order to display valued information, a Leaflet map showing store location and pricing was added to the Dashboad.

For the markers to be displayed, coordinated were needed. So instead of adding coordinates here, we used Google API during the scrape to collect coordinates of competitor store and then refrance them here.

Key Challanges: We created a custom icon, but were having a problem keeping the flag at its position as we zoomed in and out

Solution: With some research and trial/error, we got it to anchor at the "Our Store" coordinate.

Graphing Information 2. D3'ed on Bars.

Along with item information, the Dashboard should also give more insight on to the business.

The only information we transfered from original database for this project was department, category, and quantity. So we were somewhat limited on information.

Key Challange: To make the graphs more interactive, and change on click or mouseover.

Solution: Trial and Error, we stuck with what worked. With more time, our focus can be towards taking sales/transaction data and displaying associated graphs.
